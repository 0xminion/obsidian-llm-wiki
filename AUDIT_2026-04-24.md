## Detailed Findings

### H1. `qmd_mcp.py:85-86` ‚Äî Session header case-sensitivity bug

```python
with urllib.request.urlopen(req, timeout=t) as resp:
    hdrs = dict(resp.headers)
    if "mcp-session-id" in hdrs:
        self._session_id = hdrs["mcp-session-id"]
```

`resp.headers` is case-insensitive (it's an `email.message.EmailMessage`), but `dict(resp.headers)` preserves the server's original casing. If QMD responds with `Mcp-Session-Id` (Mixed-Case) instead of `mcp-session-id` (lowercase), the membership test fails and session is never persisted. **Impact:** every `_call()` re-initializes a new MCP session, adding 1-2 roundtrips per query.

**Fix:** Use `resp.headers.get("mcp-session-id")` instead of `dict()` conversion.

---

### H2. `compile.py:465-468` ‚Äî Dead availability check skips local embeddings

```python
from pipeline.qmd_mcp import _get_qmd_client
if _get_qmd_client() is not None:
    log.info("QMD enabled ‚Äî skipping local embed batch; relying on QMD semantic search")
    return
```

`_get_qmd_client()` always returns a `QMDMCPClient`. It never performs health check nor returns `None`. Thus:
1. `if client is not None` is always True.
2. Local Ollama batch embedding is **always skipped**, even when QMD is down or unreachable.
3. `NoteIndex.embeddings` remains empty, causing `_semantic_crosslink`, `_semantic_concept_merge`, and `_semantic_moc_rebuild` to silently use zero-similarity heuristics.

**Fix:** Use `pipeline.qmd._get_client()` (which performs health check and returns `None` on failure) instead of `_get_qmd_client()`.

---

### M1. `compile.py:707` ‚Äî Discarded `re.sub` result

```python
re.sub(r"^---\s*\n.*?\n---\s*\n", "", canonical_content, flags=re.DOTALL)
duplicate_body = re.sub(r"^---\s*\n.*?\n---\s*\n", "", duplicate_content, flags=re.DOTALL)
```

The first `re.sub` result is not assigned to a variable. `canonical_body` is never extracted. The merge logic later appends `duplicate_body` to the full `canonical_content`, which retains its frontmatter. This works today but is a maintenance hazard.

**Fix:** Assign to `canonical_body`.

---

### M2. `store.py:44-72` ‚Äî Thread-safety design weakness

`_LockedConnection` guards only `execute`, `executescript`, `commit`, `close`. `__getattr__` proxies everything else to the raw `sqlite3.Connection`, allowing unguarded access like `.cursor()`. Additionally, a single `sqlite3.Connection` shared across threads with `check_same_thread=False` is not recommended by SQLite docs; WAL mode with RLock helps but does not eliminate all race conditions.

**Fix:** Either use `sqlite3.connect(..., check_same_thread=True)` with a ThreadPool-local strategy, or explicitly serialize all access through the `_LockedConnection` proxy.

---

### M3. `qmd_mcp.py:225-229` ‚Äî `_get_qmd_client()` does not perform health check

```python
def _get_qmd_client(base_url: str = "") -> QMDMCPClient:
    url = base_url or _DEFAULT_QMD_MCP_URL
    return QMDMCPClient(base_url=url, timeout=60)
```

Always returns a constructed client regardless of server reachability. Callers expecting `None` for unreachable servers are misled.

---

### M4. `qmd_mcp.py:114-118` ‚Äî Missing explicit encoding on health decode

```python
with urllib.request.urlopen(req, timeout=5) as resp:
    return json.loads(resp.read().decode())
```

`.decode()` without `utf-8` argument defaults to `sys.getdefaultencoding()`, which may not be UTF-8 on certain systems.

**Fix:** `.decode("utf-8")`.

---

### M5. `qmd.py:95-155` ‚Äî Serial queries in `run_qmd_concept_search`

The function iterates `for h, q in queries.items()` and makes one `_query_raw` call per query. With `PARALLEL=3` and 20-50 sources, this serializes QMD calls, losing the parallelism of the previous embedding batch approach. Each call is ~200-500ms (HTTP + search); N=50 ‚Üí 10-25s sequential.

**Fix:** Use `ThreadPoolExecutor(max_workers=cfg.parallel)` to batch QMD queries in parallel, matching the parallelism level of the rest of Stage 2.

---

### L1. `llm_client.py:165-167` ‚Äî Silent 404 on old Ollama

```python
except urllib.error.HTTPError as e:
    if e.code == 404:
        log.debug("Ollama /api/embed not available (404)")
```

Debug-level logging means operators never see the message. `embed_batch` returns `{}`, and callers like `NoteIndex.embed_all()` degrade to heuristics with only a generic warning.

**Fix:** `log.warning` with actionable guidance (upgrade Ollama or use QMD MCP).

---

### L2. `vault.py:324` ‚Äî Edge cache not canonicalizing symmetric edges

`write_edge()` stores `(source, target, type.value)` directly. `_edge_key()` in `compile.py:218-226` sorts `RELATES_TO` pairs for idempotency. This means two symmetric vault writes become two TSV rows, while compile deduplication sees them as one. Minor inconsistency; compile cleans up.

**Fix:** Use `_edge_key()` when computing the `edge_key` in `write_edge()` cache check.

---

### L3. `qmd.py:106-114` ‚Äî Redundant lex fallback

`qmd_mcp.py:190-213` already implements vec ‚Üí lex fallback inside `client.query()`. Line 106-114 re-implements the same fallback. The second call is dead code.

**Fix:** Remove the redundant lex block; let `client.query()` handle fallback.

### L6. `extractors/web.py:299` ‚Äî Camoufox SPA wait strategy

```python
await page.goto(url, wait_until="domcontentloaded", timeout=timeout * 1000)
await asyncio.sleep(3)
```

`domcontentloaded` fires when the DOM tree is built, not when JS-populated content has rendered. For SPAs (e.g. Medium's dynamic loading), `domcontentloaded` + fixed 3s sleep may be insufficient. Some pages may still hold Cloudflare challenge content.

**Fix:** Use `wait_until="networkidle"` or `await page.wait_for_load_state("networkidle")` after the 3s sleep, with a short additional timeout.

## Security Analysis (bandit)

| Severity | Class | Count | Key Files |
|----------|-------|-------|-----------|
| **High** | B324 (MD5 `usedforsecurity=False`) | 23 | `config.py`, `models.py`, `utils.py`, `vault.py`, `store.py`, `_common.py` + tests |
| **Medium** | B314 (XML ElementTree unsafe) | 2 | `extractors/podcast.py:603-604` |
| **Medium** | B310 (urllib urlopen) | 5 | `llm_client.py` x4, `qmd_mcp.py` x2 |
| **Medium** | B108 (hardcoded /tmp) | 3 | `config.py`, test files |

### Assessment
- **B324 HIGH**: MD5 is used for URL hash and content dedup, **not for security**. This is a false positive but bandit is correct to flag it. **Recommendation**: Add `usedforsecurity=False` to all `hashlib.md5()` calls to suppress the lint and prevent future audit fatigue.
- **B314 MEDIUM**: `extractors/podcast.py` uses `ET.fromstring(xml_text)` with a custom parser but no entity-expansion limits. A malicious RSS feed with XML bomb/billion laughs could exhaust memory. Despite the `<!DOCTYPE` guard at line 600, a crafted feed with deeply nested tags or DTD-less entities could still exploit this. **Recommendation**: Add `parser = ET.XMLParser(resolve_entities=False)` or use `defusedxml`.
- **B310 MEDIUM**: urllib calls in `llm_client.py` and `qmd_mcp.py` are to user-configured local endpoints (`localhost:11434`, `localhost:8181`) or known API endpoints (`openrouter.ai`). Low risk.
- **B108 MEDIUM**: Hardcoded `/tmp` paths are used for ephemeral extraction. Configurable via `PIPELINE_TMPDIR`. Low risk.

### 2. README Outdated ‚Äî Embedding Architecture

The README (line 102, 213-214, 300) still states:

> "Stage 2: Plan ... Semantic concept search ‚Äî qmd embeddings (Qwen3-Embedding-0.6B-Q8)"
> "Uses qmd with Qwen3-Embedding-0.6B-Q8 for concept matching"

This is **incorrect** for QMD MCP mode. QMD MCP uses the QMD server's own embedding model (configured server-side). The `qwen3-embedding:0.6b` model is used only in **Ollama fallback mode** or via `llm_client.py` for other operations. The README incorrectly conflates two different embedding systems.

### 3. README Missing QMD MCP Documentation

No mention of `USE_QMD_MCP`, `QMD_MCP_URL` in README. `.env.example` includes them (good). Users won't know QMD MCP is the default search mechanism.

### 4. ARCHITECTURE.md Describes Old Embedding Flow

Section 8 (QMD Semantic Search) still describes Ollama-local embedding flow as the primary path. Needs update to describe QMD MCP HTTP transport as the default.

### 5. `.env.example` Missing `LLM_PROVIDER` Documentation

The `.env.example` mentions `LLM_PROVIDER` but README's "LLM Provider Configuration" section is more detailed but doesn't mention the `LLM_BASE_URL` or `LLM_TIMEOUT` overrides.

3. **Synchronize versions** across `pyproject.toml`, `qmd_mcp.py`, and add a version header to `README.md`.
4. **Update README + ARCHITECTURE** to describe QMD MCP as the default semantic search, with Ollama as fallback.

6. **Parallelize QMD queries** in `run_qmd_concept_search` with `ThreadPoolExecutor`.
7. **Add test coverage** for the dead-code path (H2) and edge cache canonicalization (L2).
